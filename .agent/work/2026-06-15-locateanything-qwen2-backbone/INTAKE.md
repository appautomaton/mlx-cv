## Objective
Implement the Qwen2.5 language backbone for LocateAnything as the first Phase 4 change: RMSNorm, SwiGLU, RoPE, GQA, KV-cache, and LocateAnything block-mask support, with tiny reference parity and without building the full multimodal VLM.

This unblocks the heavy LocateAnything path while keeping the next execution slice bounded: prove the text decoder and generation-critical mechanics before adding MoonViT, projector/image-token scatter, tokenizer/chat template, or PBD generation.

## Constraints
- `core/` stays mlx-free. Qwen2 implementation lives under `backbones/llm/qwen2/` behind the `[mlx]` extra.
- No `torch` or `transformers` dependency in `pyproject.toml`; reference execution is only for out-of-band fixture minting under `tools/`.
- Preserve Stage-1 LocateAnything config/convert/decode imports without requiring `mlx`.
- Do not broaden into MoonViT, multimodal projector, image-token scatter, tokenizer, chat template, PBD generate, or full LocateAnything end-to-end parity.
- Prefer reference-compatible parameter names so existing `language_model.model.*` keys from `models/locateanything/convert.py` remain useful.

## Scope Coverage
- Included: canonical Qwen2 config binding, Qwen2 model body, tied logits, RMSNorm/SwiGLU/RoPE/GQA, 4D additive masks, block-diff mask helpers, append-only KV-cache, reference fixture minting, convert/load rules, registry, and package guardrails.
- Deferred: MoonViT, projector, image-token scatter, processor/tokenizer, PBD generate loop, Magi/FlashAttention backends, training loss breakdowns, sliding-window cache, quantization, real 3B checkpoint loading.
- Anti-goals: changing DA3, hardening camera pose/intrinsics, downloading hub weights, adding runtime torch/transformers dependencies, implementing full VLM inference.

## Recommended Approach
Build from the decoder leaves upward: config and public contract -> leaves -> attention/masks -> decoder/model body -> cache -> convert/parity -> integration guardrails. Keep every slice independently verifiable, and use tiny fixtures for numerical proof.

## Key Assumptions and Risks
- LocateAnything's Qwen2 differs from stock Qwen2 mainly in `causal_attn=False`, `block_size=6`, `text_mask_token_id`, and block-diff mask preparation. Those semantics are part of this change.
- Cache parity is the highest-risk runtime behavior because RoPE position IDs, mask width, and repeated KV heads must agree across full-sequence and one-step decode.
- The local Stage-1 converter currently drops `language_model.lm_head.weight` as tied. The Qwen2 implementation should make tied logits explicit so this remains correct.
