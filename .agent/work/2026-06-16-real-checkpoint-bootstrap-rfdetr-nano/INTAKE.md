## Objective

Reorganize the roadmap around real checkpoint-backed validation first, then frame the next phase around RF-DETR Nano as the first real pretrained checkpoint bootstrap.

## Broader Intent

Stop accumulating plausible local MLX inference pipelines that only pass tiny/local fixtures. The project needs at least one real checkpoint load/run/parity path before expanding into DA3 multi-view, SAM video/Object Multiplex, or a new model family.

## What Makes This Cool

The first successful real-checkpoint path turns `mlx-cv` from architecture plumbing into something that can prove it runs actual pretrained models. RF-DETR Nano is small enough to be the proof point without dragging in the harder licensing and tap-capture problems first.

## Closure Target

Reference system: upstream RF-DETR Nano COCO checkpoint and reference detector, compared against the local MLX RF-DETR implementation.

Closed means the checkpoint is fetched or supplied outside git, checksum-verified, run through upstream and MLX paths on the same fixed input, compared for outputs/taps within tolerance, and recorded as `UPSTREAM_PASSED` only after a real pass. A blocker is not an acceptable exit for this bootstrap phase.

## Constraints

- Raw upstream checkpoints and converted model weights must stay outside git.
- Download/cache/checksum behavior must be explicit and reproducible.
- Package runtime must remain free of Torch/reference dependencies and imports from `references/`.
- RF-DETR Nano is the first target because it has a concrete URL, expected MD5, permissive Apache-2.0 weight status for Nano-Large, and existing local MLX scaffolding.
- Existing verified release-parity hardening artifacts remain truthful: they record blockers, not upstream passes.
